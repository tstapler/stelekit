# Implementation Plan: demo-graph-isolation

**Feature**: Isolate demo graph from real user graphs using an `isDemo` flag and in-memory backend
**Date**: 2026-07-05
**Status**: Ready for implementation
**ADRs**: ADR-017-demo-graph-isolation-strategy.md

---

## Domain Glossary

| Term | Definition |
|---|---|
| **Demo Graph** | A read-only-by-convention, in-memory graph loaded during onboarding to showcase app features; identified by `GraphInfo.isDemo = true` and `GraphId("__demo__")` |
| **Real Graph** | A user-owned knowledge base backed by a SQLite database and a directory on the local or OPFS filesystem |
| **Graph Registry** | The `GraphRegistry` JSON blob persisted in `platformSettings["graph_registry"]`; the single source of truth for known graphs across restarts |
| **isDemo flag** | `GraphInfo.isDemo: Boolean = false`; when `true`, the entry is ephemeral — never written to the registry and stripped on load |
| **DemoFileSystem** | A generated `FileSystem` implementation backed by hardcoded Kotlin string literals; currently in `wasmJsMain`, to be moved to `commonMain` |
| **IN_MEMORY backend** | `GraphBackend.IN_MEMORY`; SQLDelight is not used; all repositories hold data in `LinkedHashMap` / `CopyOnWriteArrayList` within the process |
| **activeGraphFileSystem** | The `FileSystem` instance that `GraphLoader` should use for the active graph; equals `DemoFileSystem` when the demo is active, `PlatformFileSystem` otherwise |
| **DEMO_GRAPH_ID** | Constant `GraphId("__demo__")` — a fixed ID that is not a sha256 path hash; stable, human-readable, guards against collision |
| **Graph Switcher** | The dropdown in `Sidebar.kt / GraphSwitcher()` that lists all real graphs; demo is omitted from this list |
| **WASM workaround** | Lines 117–121 of `kmp/src/wasmJsMain/kotlin/dev/stapler/stelekit/browser/Main.kt` that clear `graph_registry` when it contains the path `/demo`; superseded by `isDemo` stripping in `loadRegistry()` |

## Pattern Decisions

| Decision | Pattern | Rationale | Rejected Alternative |
|---|---|---|---|
| **How to mark a demo graph** | Structural field `isDemo: Boolean = false` on `GraphInfo` (Type-Driven Design) | Propagates the distinction through the type system; checked at registry load, switchGraph, rename; zero string comparisons | ① Sentinel `GraphId("__demo__")` alone — detectable but not type-safe (callers must remember to check); ② `sealed class GraphKind` hierarchy — correct but requires breaking every `GraphInfo` call site |
| **Where DemoFileSystem lives** | `commonMain` (generated by Gradle task; output path changed from `wasmJsMain`) | JVM and WASM both need it; one class, one generator run, identical content | Keep in `wasmJsMain` and add a JVM `expect/actual` — doubles the generated file and diverges content |
| **activeFileSystem threading** | `remember(activeGraphInfo?.isDemo)` in App.kt derives `effectiveFileSystem` from registry state; no new StateFlow on GraphManager | Keeps GraphManager's filesystem concern local; App.kt already observes `graphRegistry` | Expose `activeGraphFileSystem: StateFlow<FileSystem>` on GraphManager — adds API surface and coupling for a purely presentation concern |
| **Onboarding → App.kt interface** | Add `onDemoSelected: () -> Unit` param to `Onboarding` alongside existing `onGraphSelected: (String) -> Unit` | Clear separation; prevents demo path from reaching `addGraph()` | Pass a sentinel string like `"__demo__"` via `onGraphSelected` — relies on string matching, same smell as current workaround |

---

## Migration Plan

No schema changes to `SteleDatabase.sq`. No `MigrationRunner` entries needed.

`GraphInfo` gains `isDemo: Boolean = false` with a Kotlin default. The existing `json = Json { ignoreUnknownKeys = true }` instance in `GraphManager` handles deserialization of registries that lack this field. Since `encodeDefaults` is **not** set, `isDemo = false` is omitted from JSON for real graphs, keeping registry payloads identical to today.

Registry entries with `isDemo = true` are stripped in `loadRegistry()` before persisting. Any real `isDemo = true` entry that somehow existed in production would be silently removed on first load — this is the desired behavior.

## Risk Control

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Demo was the only graph; stripping it in `loadRegistry` leaves `graphs = emptyList()` | Low (only demo existed) | High (loading overlay hangs) | After stripping, if `graphs.isEmpty()` OR `activeGraphId == null`, ensure `onboardingCompleted = false` is cleared in `platformSettings` so App.kt routes to Onboarding (line 1147 check already does this) |
| `remember(activeGraphInfo?.isDemo)` recomposition creates a fresh `DemoFileSystem` | Negligible (only on isDemo toggle) | Low | DemoFileSystem is stateless; recreation is safe; `remember` key prevents re-creation on unrelated recompositions |
| Generator output path change breaks incremental Gradle build cache | Low | Medium | `inputs.dir` and `outputs.file` are declared; Gradle will invalidate cache when output path changes; first build after the change regenerates cleanly |
| `DemoFileSystem` in `commonMain` pulls in `kotlin.time.Clock` / `kotlinx.datetime` which must be in commonMain | None — both already present | — | Verified via existing `commonMain` dependency declarations in `build.gradle.kts` |
| `DEMO_GRAPH_ID = "__demo__"` collides with a real graph sha256 hash | Astronomically unlikely | Medium | `sha256("any path").take(16)` is hex; `"__demo__"` contains underscores — format mismatch makes collision structurally impossible |

## Phase 2: GraphManager

### Epic 2.1: Demo-aware graph lifecycle in GraphManager

**Goal**: `addDemoGraph()` creates an ephemeral in-memory-only entry; `loadRegistry()` strips it on restart; `switchGraph()` forces IN_MEMORY; `renameGraph()` rejects it.

#### Story 2.1.1: Add `addDemoGraph()` method

**As a** caller in `App.kt`, **I want** to register a demo graph with a single call that does not touch the filesystem or persistent registry, **so that** the demo leaves no trace on disk.

**Acceptance Criteria**:
- Calling `addDemoGraph()` adds a `GraphInfo(id = DEMO_GRAPH_ID, path = "/demo", displayName = "Demo Graph", isDemo = true)` to `_graphRegistry.value` but does NOT call `saveRegistry()`.
- The registry stored in `platformSettings["graph_registry"]` is unchanged after calling `addDemoGraph()` (demo entry is in-memory only). **Mechanism**: persistence is prevented at the `saveRegistry()` layer (Task 2.1.1b), which filters demo entries before writing, so ALL 11 call sites — including `switchGraph()` which unconditionally calls `saveRegistry()` — are covered without per-call-site guards.
- Calling `addDemoGraph()` twice is idempotent (no duplicate in `registry.graphs`). The idempotency check uses `_graphRegistry.update {}` for atomic CAS semantics (Task 2.1.1a), preventing races if called concurrently.
  - *Given* no prior call to `addDemoGraph`, *When* `addDemoGraph()` is called, *Then* `graphRegistry.value.graphs` contains exactly one entry with `id == DEMO_GRAPH_ID` and `isDemo == true`, AND `platformSettings.getString("graph_registry", "")` does not contain `"__demo__"`.
  - *Given* `addDemoGraph()` was already called, *When* `addDemoGraph()` is called again, *Then* `graphRegistry.value.graphs.count { it.id == DEMO_GRAPH_ID } == 1`.
  - *Given* `switchGraph(DEMO_GRAPH_ID)` is called (which unconditionally calls `saveRegistry()`), *When* `saveRegistry()` runs, *Then* `platformSettings.getString("graph_registry", "")` does not contain `"__demo__"` and `activeGraphId` in the saved JSON is null (not `"__demo__"`).

**Files**:
- `kmp/src/commonMain/kotlin/dev/stapler/stelekit/db/GraphManager.kt`

##### Task 2.1.1a: Implement `addDemoGraph()` (~5 min)
- Add `fun addDemoGraph(): GraphId` after the `addGraph` function (around line 272).
- Construct `GraphInfo(id = DEMO_GRAPH_ID, path = "/demo", displayName = "Demo Graph", addedAt = Clock.System.now().toEpochMilliseconds(), isDemo = true)`.
- Use `_graphRegistry.update { registry -> if (!registry.graphIds.contains(demoId)) registry.copy(graphs = registry.graphs + info) else registry }` for atomic CAS semantics (prevents duplicate insertion if called concurrently from multiple coroutines). Do NOT use the non-atomic `val registry = _graphRegistry.value; if (...) { _graphRegistry.value = registry.copy(...) }` pattern.
- Do NOT call `saveRegistry()`.
- Do NOT call `checkGitignoreForDatabase`, `detectGitRoot`, or `fileSystem.expandTilde`.
- Return `DEMO_GRAPH_ID`.
- Files: `kmp/src/commonMain/kotlin/dev/stapler/stelekit/db/GraphManager.kt`

##### Task 2.1.1b: Guard `saveRegistry()` against demo entries (~3 min)
- In `saveRegistry()`, replace:
  ```kotlin
  platformSettings.putString("graph_registry", json.encodeToString(_graphRegistry.value))
  ```
  with a filtered snapshot that strips demo entries before writing:
  ```kotlin
  val toSave = _graphRegistry.value.let { r ->
      r.copy(
          graphs = r.graphs.filter { !it.isDemo },
          activeGraphId = if (r.activeGraphId == DEMO_GRAPH_ID) null else r.activeGraphId
      )
  }
  platformSettings.putString("graph_registry", json.encodeToString(toSave))
  ```
- This ensures demo entries never reach disk regardless of which of the 11 call sites triggers the save (including `switchGraph()`, which unconditionally calls `saveRegistry()`). No per-call-site guards are needed.
- Files: `kmp/src/commonMain/kotlin/dev/stapler/stelekit/db/GraphManager.kt`
