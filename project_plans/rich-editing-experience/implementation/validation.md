# Validation Plan: rich-editing-experience

**Date**: 2026-07-05

## Happy Path Scenario

Given the confirmed-broken baseline where Ctrl+Enter falls through to plain-Enter (splitting the block instead of toggling TODO state), when the user places the cursor in a block with content `"Call mom"` and presses Ctrl+Enter with no autocomplete popup open, then `handleKeyEvent` dispatches `requestTodoToggle()` → `applyTodoToggle` parses `TodoState.NONE`, prepends `TODO `, and the block's content becomes `"TODO Call mom"` — the single fix (Phase C.1) that anchors every later phase's "does the new plumbing actually reach the block" pattern (mobile toolbar button, command-palette entry, shortcut table lookup all reuse this exact dispatch shape).

## Test Stack

- **Unit**: Kotlin/JUnit — `kotlin.test` assertions (`assertEquals`/`assertTrue`/`assertFalse`) inside `org.junit.Test`-annotated methods, matching `KeyboardShortcutTest.kt`'s existing style. Pure-logic tests (`TodoState.parse`/`.next()`, `applyFormatAction`, `applyTodoToggle`, backlog/journey-doc schema parsing) live in `kmp/src/jvmTest` (if they touch Compose types like `TextFieldValue`) or `kmp/src/businessTest` (if pure Kotlin with no UI dependency, per `kmp/TESTING_README.md`'s source-set split).
- **Integration**: Compose UI testing (`createComposeRule()`, `androidx.compose.ui.test.*`, `ExperimentalTestApi`) exercising multiple composables together (`EditorToolbar` + `BlockItem` + `TagChipRow`, `CommandPalette` + `StelekitViewModel` + `BlockStateManager`, `AnnotationEditorScreen` + `AnnotationToolbar`) — this project has no repository/DB-boundary changes, so "integration" here means cross-composable wiring, not database integration. Doc-schema cross-checks (`gap-backlog.md` ↔ `docs/journeys/*.md` ↔ `plan.md` reconciliation) are plain-Kotlin file-parsing tests, no framework needed beyond `kotlin.test`.
- **E2E / UX**: Roborazzi screenshot tests (`kmp/src/jvmTest/kotlin/dev/stapler/stelekit/ui/screenshots/MobileScreenshotTest.kt`, `DesktopScreenshotTest.kt`) for visual regression, gated on Epic D.0's go/no-go spike (manual visual-QA checklist fallback if NO-GO). Manual/instrumented device testing is used — not forced into a Playwright framing — for: Android/iOS long-press drag-to-reorder gestures, iOS hardware-keyboard shortcut verification (Task E.3.1c), CJK IME composition testing (Task E.1.1d, JVM/desktop tests cannot reproduce real IME composition), and Web browser-reserved-shortcut suppression (Task E.2.1a, requires an actual browser, not a JVM test harness).
