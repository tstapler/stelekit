# Fix Android Build (BUG-007) (Feb 22, 2026)

## Objective
Resolve compilation errors in `commonMain` caused by Desktop-specific pointer events that are unavailable on Android.

## Analysis
The `App.kt` file in `commonMain` was using `onPointerEvent` and `PointerButton` (Desktop-specific APIs) to handle mouse back/forward button navigation. This caused the Android build to fail since these APIs don't exist in the Android Compose runtime.

## Resolution
1. **Abstract Navigation Input**: Created an `expect` extension function `Modifier.platformNavigationInput(onBack: () -> Unit, onForward: () -> Unit)` in `commonMain`.
2. **JVM Implementation**: Implemented the `actual` version in `jvmMain` using the original `onPointerEvent` logic for mouse buttons.
3. **Android Implementation**: Implemented an empty `actual` version in `androidMain` since mouse navigation is not typical for mobile.
4. **Update App.kt**: Refactored `LogseqApp` to use the new platform-agnostic extension.

## Files Modified
- `kmp/src/commonMain/kotlin/com/logseq/kmp/ui/ModifierExtensions.kt` (New)
