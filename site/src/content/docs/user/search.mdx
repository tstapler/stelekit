---
title: Search
description: Full-text search across all pages and blocks in SteleKit
---

Press **Cmd+K** (macOS) or **Ctrl+K** (Windows/Linux) to open search from anywhere in the app.

## What gets indexed

Search covers:
- Page titles
- All block text on every page
- Journal entries
- Property values

Search does not cover file attachments or images.

The index builds when you open a graph and stays current as you edit. New blocks appear in results without a manual refresh.

## Using search

1. Press Cmd/Ctrl+K to open the search dialog
2. Start typing — results appear as you type
3. Use the arrow keys to move through results
4. Press Enter to navigate to the selected result
5. Press Escape to close without navigating

The search dialog closes automatically after you navigate to a result.

## Result format

Each result shows:
- The page name
- The matching block with the search terms highlighted
- Enough surrounding context to identify which part of the page matched

## Navigating to a result

Selecting a result opens the page and scrolls to the matching block. The block is briefly highlighted so you can find it on the page.

## Search scope

Search covers all open graphs simultaneously. Results are grouped by graph when you have more than one open.

## Tips

- Search for a partial word — results match substrings
- Search page names to jump to a page by title without using the sidebar
- Use Cmd/Ctrl+K as the primary navigation tool; it is faster than clicking the sidebar for pages you know by name
