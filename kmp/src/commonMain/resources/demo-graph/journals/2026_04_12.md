- Weekly review. Five days with SteleKit — time to consolidate what I learned.
- **What I learned this week**
	- Block model
		- [[Block Editing]] — blocks are the atomic unit; Tab/Shift+Tab handle indentation naturally
		- Nesting is more expressive than I expected; collapse/expand changes how I read long pages
	- Linking and graph
		- [[Page Linking]] — `[[` shortcut and suggestion dropdown make linking effortless
		- Backlinks appear without any extra work; the graph grows as a side effect of writing
	- Structured data
		- [[Properties]] — `key:: value` syntax on page and block level; no schema needed
		- Consistent key naming is important from the start (lowercase convention)
	- Spaced repetition
		- [[Flashcards]] — `card:: true` property on any block adds it to the review queue
		- Atomic, short cards recall better than long blocks
- **What worked well**
	- Daily journal as inbox — capture first, file later. No friction at capture time.
	- Links instead of folders — the graph structure emerged naturally without planning a hierarchy.
	- The demo graph pages (this graph!) are useful reference material while learning.
- **Plans for next week**
	- Go deeper on [[Search]] — learn how to surface blocks by content, not just page titles.
	- Explore [[Markdown Formatting]] — want to use fenced code blocks for technical notes.
	- Set up a consistent property vocabulary for my research pages.
	- Keep the flashcard habit going: review queue every morning.
- One week in, SteleKit is already replacing two other apps for me. The local-first, Markdown-native approach feels right.
